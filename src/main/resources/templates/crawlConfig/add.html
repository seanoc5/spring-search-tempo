Pattern Configuration (Optional)
Pattern Hierarchy

Files and folders are matched against patterns in order of priority:

  1. SKIP - Metadata only (no children enumerated for folders, no text extraction for files)
  2. LOCATE - Store metadata only (path, size, timestamps)
  3. INDEX - Extract full text and metadata
  4. ANALYZE - INDEX + sentence-level chunking + NLP

Use Java regex syntax. One pattern per line. Global defaults (from application.yml) are applied if no patterns are specified.
