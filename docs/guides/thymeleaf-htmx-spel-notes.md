# Thymeleaf + HTMX + SpEL Notes

Practical rules for templates in `src/main/resources/templates`.

## 1) HTMX Event Attributes With Thymeleaf

Use different syntax depending on whether the attribute is static HTML or generated by `th:attr`.

- Static HTML attribute (safe):
  - `hx-on::after-request="handleDone(event)"`
- Dynamic attribute via `th:attr` (safe):
  - `th:attr="hx-on--after-request=|handleDone(${id})|"`
- Dynamic attribute via `th:attr` (will break parse):
  - `th:attr="hx-on::after-request=|handleDone(${id})|"`

Reason: `th:attr` parses an assignation sequence (`name=value`). Keys with `::` are not parsed correctly there.

## 2) Prefer Dedicated `th:hx-*` Processors

When possible, prefer:

- `th:hx-get`
- `th:hx-post`
- `th:hx-target`
- `th:hx-swap`

Only use `th:attr` for attributes without dedicated processors or for grouped assignment.

## 3) `th:attr` Safety Rules

- Use valid assignations: `th:attr="name=value, name2=value2"`
- Use literal substitution for mixed text + expressions:
  - `th:attr="data-label=|Run #${run.id}|"`
- Keep complex expressions short by moving pieces to `th:with`.

## 4) SpEL/Expression Hygiene

- Guard nullable values explicitly in expressions:
  - `${obj != null ? obj.value : '-'}`
- Avoid deep chained lookups without checks.
- Use `th:with` for repeated computed values in a block.

## 5) Common Parse-Error Triage

If you see:

- `Could not parse as assignation sequence`

Check first:

1. Any `th:attr` key using `::` (especially `hx-on::...`)?
2. Missing commas between `th:attr` assignments?
3. Broken quote or pipe-literal in `|...|` strings?
4. Unbalanced parentheses in `${...}`?

## 6) Fast Validation Loop

1. Run targeted web tests:
   - `./gradlew test --tests com.oconeco.spring_search_tempo.web.controller.DashboardIntegrationTest`
2. Reload affected page and watch server logs for template parse errors.
3. Search for risky patterns:
   - `rg -n 'th:attr="[^"]*hx-on::' src/main/resources/templates`
